Variables

In Javascript there are different ways of declaring a variable like let, var and const in this course we use const and sometimes we may use let. Defining a variable in Javascript is easier compared to other languages like Java. We don't use var because it's global meaning when defined in a scope it can also be used outside a scope. For example if you define const name = "Samim" inside a function you can use it outside a function but if you use let or const it won't be accessable outside the scope.

We don't need to define what data type it is. We can just create a variable like this const name = "Samim".

Objects

Objects is a way of storing a collection of key-value pairs. Below is an example of an product object. It has 2 key-value pairs. It has name and the name of the product and the price of the product. 

            
  const product = {
    name: "Macbook Pro",
    price: 9900
  }

Arrays

Arrays is also a way of storing values but instead of a key-value pair it only stores a single variable. Below is an example of how to declare an array of product categories. The array is split in elements and you are able to push and pop elements from the array. 

            
  const categories = ["Laptops", "Phones", "Tablets"]

Functions

Functions is a way to write code that is reusable. There are different way to create a function below is examples of the different ways to create functions. With function you can call the same code as many times as you want and you are also able to return something. 

            
  // Normal function
  function getRandomInt(min, max) {
    return Math.floor(Math.random() * (max + 1 - min) + min);
  }

  // Anonymous Function
  const getRandomIntAnonymous = function(min, max) {
    return Math.floor(Math.random() * (max + 1 - min) + min);
  }

  // Arrow Function
  const getRandomIntArrow = (min, max) => {
    return Math.floor(Math.random() * (max + 1 - min) + min); 
  }

  // Oneliner arrow function
  const getRandomIntArrowOneliner = (min, max) => Math.floor(Math.random() * (max + 1 - min) + min); 

  // Generic Action Performer
  function genericActionPerformer(name, genericAction) {
    return genericAction(name);
  }

Loops

In Javascript there are many way to loop through a list. In this course we mostly use map, filter and reduce as some loops can create unnecessary side effects so we should try avoiding it. Below is an example of how we map through a list of products and increase the prices of the product by 10% 

            
  const increasedPrices = products.map((product) => {

  });

Asynchronous

Asynchronous code allows us to execute code without cancelling the other code. This is useful when you have to wait for something like when you fetch data from an API it can take some time before you get the data. That's why we use asynchronous to avoid waiting for it to finish before we continue.

One way of having asynchronous code is using promises. A promise can have two states, either pending or fulfilled which can result in it either being resolved or rejected below is an example of how you would create a promise that returns Everything OK after 2 seconds. 

            
  new Promise((resolve, reject) => {
    setTimeout(() => {
      try {
        resolve("Everything OK")
      } catch (error) {
        reject("Nothing OK")
      }
    }, 2000);
  })
  .then((successMessage) => console.log(successMessage))
  .catch((errorMessage) => console.log(errorMessage));
Another way of having asynchronous code is using async / await. We can have a async function and then you can await data. Below is an example of how a async / await function would look like. 
            
  async function main() {
    try {
      const myPromiseResult = await myPromise();
    } catch (error) {
      console.log(error);
    }
  });
