Express

Express is a Node.js web application framework. Express makes it easier for developers to build webapps. In this course we use Express to serve HTML files and create API'. Express has many built-in features like routing (e.g. GET, POST, PUT and DELETE) and we are able to send files to the webbrowser using sendFile(). Express is a great tool when working with building web apps for both front- and backend.

REST API

REST API is a web service that makes it possible to communicate between application and computers. It is something that you would find in most web applcations. Below is an representation of how REST API works, the client sends a request to the server which usually calls a database and then uses one of the HTTP methods like GET. There are some principles we should remember when working with REST API. It is how we define our endpoints as we've learned in the course to have some standards when making an API. Below is a table of how an REST API structure would look like. Let's use products as an example. We don't use /getProducts or /editProduct. We should only use /products for all HTTP methods.

HTTP Methods Endpoints Explanation
GET /products Get all products
GET /products/{id} Get product by ID
POST /products Create a product
PUT /products/{id} Edit a product resource by ID
PATCH /products/{id} Edit a part of product by ID
DELETE /products/{id} Delete a product by ID

Routers

Routers is a way to organize your code. Instead of having all your endpoints in a single file, we can create routers which is implemented in Express. Below is an example of how you would implement it. 

            
  import { Router } from 'express';
  const router = Router();

  router.get("/", (req, res) => {
    res.send(homePage);
  });
We create a routers in the project folder which contains all the http methods for that specific category. It could be a pagesRouter and a productsRouter.
